# Тестируем

## Кнок

Ключ лицензии представляет собой последовательность портов, например: U00033-U00044-U10000-U20000-U00444. Каждый символ "U" указывает на тип протокола, в данном примере выполняется кнок по портам UDP: 33 -> 44 -> 10000 -> 444.

Чтобы просмотреть логи кнока, используйте комбинацию клавиш CTRL+SHIFT+F11.

## Адресная книга

Для удобства хранения контактов, в самом приложении реализован веб-сервер, который слушает адрес http://127.0.0.1:9999/contacts.xml. Вы можете открыть его в браузере. Само приложение MicroSIP настроено для получения контактов с этого адреса.

Контакты также могут быть загружены с адреса https://sipserver:9443/contacts.xml. В этом случае сертификат игнорируется, поэтому можно использовать самоподписанный сертификат.

Обратите внимание, что в данном случае реализуется некий прокси: мы получаем контакты с https://sipserver:9443/contacts.xml и передаем их по адресу http://127.0.0.1:9999/contacts.xml. Подготовьте адресную книгу в соответствии с требованиями MicroSIP.

Обновление контактов происходит каждые 10 минут, а также можно выполнить обновление вручную с помощью комбинации клавиш CTRL+SHIFT+F10.

## Настройки

При запуске программы происходит проверка наличия соответствующих настроек в реестре по адресу `HKEY_CURRENT_USER\SOFTWARE\MicroSIP-Wrapper`:

```
[HKEY_CURRENT_USER\SOFTWARE\MicroSIP-Wrapper]
"Lisense"="U00033-U00044-U10000-U20000-U00444"
"Server"="sip.server.ru"
"Name"="Васильев Кирилл"
"Number"="1338"
"Password"="passwordsip"
```

Если настройки не найдены, программа отображает окно с требованием указать настройки.

Если необходимо изменить настройки с помощью форм, используйте комбинацию клавиш CTRL+SHIFT+F12.
